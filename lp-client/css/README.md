# ARROW pro - CSS アーキテクチャ

## 📁 ディレクトリ構造

```
css/
├── main.css                 # メインエントリーポイント（全てをインポート）
├── mobile.css              # レガシーサポート（画面サイズ別の詳細スタイル）
│
├── base/                   # 基礎レイヤー
│   ├── _reset.css         # リセットCSS & normalize
│   ├── _variables.css     # CSS変数（デザイントークン）
│   └── _typography.css    # フォント & タイポグラフィ
│
├── layout/                 # レイアウトレイヤー
│   ├── _grid.css          # グリッドシステム & 画面レイアウト
│   ├── _header.css        # ヘッダー
│   └── _footer.css        # フッター
│
├── components/             # コンポーネントレイヤー
│   ├── _buttons.css       # ボタン
│   ├── _cards.css         # カード（特徴、事例など）
│   └── _faq.css           # FAQ アコーディオン
│
├── pages/                  # ページレイヤー
│   └── _sections.css      # ページセクション（Hero, Features, etc.）
│
└── old/                    # バックアップ（旧ファイル）
    ├── custom.css
    ├── globals.css
    └── styleguide.css
```

## 🎯 設計思想

### モダンなコンポーネントベースアーキテクチャ

1. **レイヤー構造**
   - 各レイヤーは独立して管理可能
   - 責任範囲が明確
   - メンテナンス性が向上

2. **レスポンシブファースト**
   - 画面サイズごとの分割ではなく、コンポーネントごとに管理
   - 各コンポーネント内でメディアクエリを使用
   - DRY原則（Don't Repeat Yourself）を実践

3. **スケーラビリティ**
   - 新しいコンポーネントを簡単に追加可能
   - ファイルサイズが適切（500-1000行/ファイル）
   - チーム開発に適した構造

## 📝 使用方法

### HTMLでの読み込み

```html
<link rel="stylesheet" type="text/css" href="css/main.css" />
```

`main.css` が全ての必要なファイルを自動的にインポートします。

### 新しいコンポーネントの追加

1. 適切なディレクトリに新しいファイルを作成
   ```
   components/_new-component.css
   ```

2. `main.css` でインポート
   ```css
   @import url('components/_new-component.css');
   ```

## 🔄 移行について

### 現在の状態

- ✅ モダンな構造を実装
- ✅ `mobile.css` をレガシーサポートとして保持
- 🔄 段階的に `mobile.css` の内容を新構造に移行可能

### 今後の推奨事項

1. **段階的な移行**
   - `mobile.css` の各セクションを段階的に新構造に移行
   - 移行完了後、`mobile.css` を削除

2. **パフォーマンス最適化**
   - CSS Minification（本番環境）
   - Critical CSS の抽出
   - ファイルの結合（必要に応じて）

3. **保守性の向上**
   - 各コンポーネントのドキュメント化
   - スタイルガイドの作成
   - Lintツールの導入（Stylelint等）

## 📊 レイヤーの詳細

### Base Layer（基礎）
- **目的**: プロジェクト全体の基礎設定
- **含むもの**: リセット、変数、タイポグラフィ
- **変更頻度**: 低

### Layout Layer（レイアウト）
- **目的**: ページの基本構造
- **含むもの**: グリッド、ヘッダー、フッター
- **変更頻度**: 低〜中

### Components Layer（コンポーネント）
- **目的**: 再利用可能なUIパーツ
- **含むもの**: ボタン、カード、FAQ
- **変更頻度**: 中〜高

### Pages Layer（ページ）
- **目的**: ページ固有のスタイル
- **含むもの**: セクションレイアウト
- **変更頻度**: 高

## 🎨 デザイントークン

カラー、フォントサイズ、スペーシングは `base/_variables.css` で管理されています。

```css
:root {
  --cerulean: #00afcc;
  --font-size-m: 16px;
  --spacing-md: 16px;
}
```
